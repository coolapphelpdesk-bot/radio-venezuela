Privacy Policy

Radio Venezuela · last updated June 18, 2026

1. Summary

Radio Venezuela (“the app”) is a free radio streaming application that lets you listen to 89 Venezuelan radio stations live. It is published by Cool App Helpdesk.

  • The developer collects no personal data beyond what you voluntarily submit through the optional feedback form. There are no user accounts and no developer-operated servers that collect telemetry or analytics.
  • All feedback you send (subject, message, and optional contact info) is transmitted to a Cloud Run endpoint and used solely to respond to your inquiry. It is not used for profiling, advertising, or any other purpose.
  • The app does not display advertisements and contains no third-party tracking SDKs.

2. Data we collect

Minimal and only what you choose to share. Radio Venezuela does not collect personal data, telemetry, identifiers, device information, IP addresses, advertising IDs, or usage analytics. There are no user accounts.

The only data transmitted to a developer-operated server is what you type into the feedback form: a subject (optional), a message (required), and a contact email or phone (optional). This feedback is sent to a Cloud Run endpoint for the sole purpose of receiving your comments and responding if you provided contact information.

If you report a non-working station from the player, the app sends an auto-generated bug report that includes the station name, ID, stream URL, device model, and Android version. This data is used only to diagnose and fix broken station streams.

5. Network activity and third parties

Radio Venezuela makes network connections to the following endpoints:

  • Radio station stream URLs — Direct connections to radio broadcasters’ streaming servers (HTTP/HTTPS). The content is provided and controlled by the respective broadcasters. The developer has no control over, and assumes no responsibility for, the content or privacy practices of these third-party streaming servers.
  • Station artwork images — Loaded from remote URLs via Coil image loader. These images are hosted on third-party CDNs controlled by the broadcasters or station directory services.
  • Feedback API — POST requests to a Cloud Run endpoint hosted at requests-847893873240.europe-west1.run.app. This is the developer’s own endpoint, used solely to receive user feedback and station reports.
  • Station list API — A Retrofit GET call to fetch the station list on launch. If the server is unreachable, the app falls back to a local bundled station list.

The app contains no advertising SDKs, no analytics SDKs, no crash-reporting SDKs, and no social or attribution SDKs. All third-party libraries used (Media3 ExoPlayer, Retrofit, OkHttp, Coil, Room) operate strictly within the scope described above and do not transmit data to the developer or to third parties beyond what is explicitly described.

6. Children’s privacy

Radio Venezuela is a general-interest radio streaming app. It is not directed at children under 13, but we recognize that minors may use it. The developer collects no personal information from any user, regardless of age, beyond what is voluntarily submitted in the feedback form.

The app contains no advertisements, no behavioral tracking, and no in-app purchases.

7. Data retention and deletion

Because the developer collects nothing beyond feedback submissions, there is nothing for us to retain or delete on our end beyond what is necessary to respond to your inquiry. Feedback submissions are kept only as long as needed to address the reported issue.

To remove locally stored data (favorites, disabled stations, preferences), clear the app’s storage in Android Settings → Apps → Radio Venezuela → Storage → Clear storage, or uninstall the app.

Data transmitted to third-party streaming servers and CDNs is governed by those parties’ own privacy policies. The developer has no control over and is not responsible for their data practices.

8. Security

The app stores preferences and cached station data in the app’s private storage, protected by Android’s built-in application sandboxing. Network requests to the feedback API are made over HTTPS. The app is distributed through the Google Play Store, which verifies the developer signing key on every update.

9. Changes to this policy

If any material aspect of how the app handles data ever changes, this page will be updated and the “last updated” date above will be bumped. Continued use after a change constitutes acceptance.
